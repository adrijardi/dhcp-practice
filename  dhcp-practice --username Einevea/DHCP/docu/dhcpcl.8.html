NAME

dhcpcl − simple, incomplete and incompatible DHCP client

SYNOPSIS

dhcpcl iface [−t timeout] [−a address] [−l lease] [-h hostname] [−d]

DESCRIPTION

Dhcpcl is a simple, incomplete and incompatible DHCP client. It negotiates a lease with a DHCP server and configures the interface iface accordingly.

OPTIONS

−t timeout

Time to wait for a server response before giving up. The default value is 64 seconds.

−a address

The IPv4 address, in dot-decimal notation, you are trying to obtain from the DHCP server.

−l lease

The lease time you are trying to obtain, in seconds. Use ’inf’ for infinite. The default time is infinite.

−h hostname

Specifies the string that will appear in the hostname option field of the DHCP datagram.

−d

Debug mode. Developers use this option to isolate users from chatty debugging outputs.

USAGE

Dhcpcl only manage and configure an interface, it doesn’t configure the DNS resolver or any other DHCP extension.

When invoked on an already configured interface, dcpcl gives an error message on standard error.

When the program receives the SIGUSR2 signal, a release of the leased address is issued and a graceful shutdown performed. If the program is abruptly killed with the SIGINT signal, no release is performed.

Dhcpcl exit code must be 0 if all the parameters were successfully obtained, 1 if there was no response from the server and -1 if there was any error.

TRACES

The program shows traces on the standard output with the following format:

#[timestamp] PID = pid.
#[timestamp] (xid) DHCPDISCOVER sent.
#[timestamp] (xid) DHCPOFFER received from ip_s (offered ip_c).
#[timestamp] (xid) DHCPREQUEST sent to ip_s.
#[timestamp] (xid) DHCPACK received: ip_c with leasing l seconds.
#[timestamp] (xid) DHCPNACK received.
#[timestamp] (xid) DHCPRELEASE sent ip_c released.
#[timestamp] SIGINT received.
#[timestamp] SIGUSR2 received.

Timestamps are the system time, according to the format defined under RFC 3339. Xid is the transaction ID in hexadecimal. Pid is the proccess ID of the client. Useful if you want to send it a signal. Ip_s and ip_c are the server and client IPv4 in dot-decimal notation. L is the lease time in seconds.

NOTES

Dhcpcl responds to the following signals:

SIGUSR2

This signal causes dhcpcl to release the current lease, bring down the interface and ends the program.

SIGINT

This signal exits the process abruptly, without releasing the leased address or bringing down the interface.
